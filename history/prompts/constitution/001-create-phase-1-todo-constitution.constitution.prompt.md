# Phase I Todo Application ‚Äì Constitution

## Purpose
This project follows Spec-Driven Development (SDD). The objective of Phase I is
to build a simple, reliable, in-memory Todo application that runs in the
command-line interface (CLI).

All development decisions must be guided by this constitution and the approved
specification. No implementation may begin until the specification, plan, and
tasks are agreed upon.

## Core Principles
1. Specification before implementation.
2. Planning before coding.
3. Tasks before execution.
4. Simplicity over complexity.
5. No features beyond Phase I scope.

## Non-Negotiable Rules
- No manual coding by a human.
- All code must be generated by Claude Code.
- No feature additions beyond the approved specification.
- No database, file system, or persistent storage.
- No web interface or GUI.

## Technical Constraints
- Programming Language: Python 3.13 or higher
- Interface: Command Line Interface (CLI)
- Storage: In-memory only using Python data structures
- Libraries: Python standard library only
- Architecture: Clear separation of concerns

## Task Entity Rules
Each Todo task must contain:
- id: Integer, auto-incremented and unique
- title: String, required
- description: String, optional
- completed: Boolean, default is false

## CLI Behavior Rules
- Menu-driven interaction using numbered options
- Clear and readable prompts
- Graceful handling of invalid input
- Continuous loop until user explicitly exits

## Quality Standards
- Clean, readable, and maintainable code
- Small, focused functions
- Predictable behavior
- No dead or unused code
